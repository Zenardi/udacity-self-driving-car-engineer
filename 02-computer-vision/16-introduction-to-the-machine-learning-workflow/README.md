## Summary

**Machine Learning Workflow Overview**
=====================================

This README file provides a summary of the key concepts and high-level components of a machine learning workflow, as introduced in the Udacity lesson "Introduction to the Machinery Workflow".

**Key Concepts**
---------------

* **Machine Learning**: A type of artificial intelligence (AI) where a computer learns to extract information from data.
* **Machine Learning Algorithm**: A program that enables computers to learn and improve their performance on a task without being explicitly programmed.
* **Data Fitting**: The process of training a machine learning algorithm using example data, which is essential for its development and improvement.

**Practical Notes**
------------------

This lesson focuses on the high-level components of a machine learning workflow, introducing the concept of machine learning algorithms and their need to be trained with example data. The German Traffic Sign Recognition Benchmark (GTSRB) dataset is used as an example task that machine learning engineers may encounter in their future careers.

## Introduction to the Machine Learning Workflow
In the lesson, we are going to learn how to think about Machine Learning problems. Machine Learning (ML) is not only about cool math and modeling but also about choosing setting the problem, identifying the client needs and the long term goals. This lesson is going to be organized as follow:
* We will practice framing Machine Learning problems by identifying the key stakeholders and choosing the correct metrics. 
* Because ML is about data, we will discuss the different challenges linked to data.
* We will also tackle how to organize your dataset when solving a ML problem to be confident that you created a model that will perform well on new data.
* Finally, we will see how you can leverage different tools to pinpoint your model's limitations.
