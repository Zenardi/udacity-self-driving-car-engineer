## Summary

**Waymo Software Stack Roles**
=====================================

This README file provides an overview of the main topic discussed in the Udacity lesson: the various roles available at Waymo across its software stack.

### Key Concepts
* **Software Stack**: Refers to the collection of software components that work together to enable a self-driving car's functionality, including perception, prediction, planning, simulation, and more.
* **Perception**: The process by which a self-driving car interprets sensor data from the environment to understand its surroundings.
* **Prediction**: The ability of a self-driving car to forecast potential future events or scenarios based on current conditions.
* **Planning**: The process by which a self-driving car determines the best course of action given its current state and predicted future events.
* **Simulation**: A virtual environment used to test and train self-driving cars in various scenarios.

### Practical Notes
No specific practical steps, code patterns, or real-world applications were discussed in this lesson. However, it is recommended that students check Waymo's job postings for available roles across the software stack.
